# 采用与边界

Urtext 被设计为可以增量采用，也可以诚实放弃。本页讲的就是
这两件事——如何从最小可能承诺开始，以及在什么条件下你应该
完全*不要*使用 Urtext。

## 增量采用

Urtext 原生基于 git 且无服务器（原则 P8）：没有守护进程、没有长时间运行的
服务、没有 workspace、没有编排模型。它确实需要 Node.js 22+ 和
npm 包（该包会构建原生 `better-sqlite3`）：

- **`cd` 进一个你已有的仓库**并写一个子句(clause)。你不需要
  转换整个项目。
- **只规范那些承重承诺。** UI 细节可以继续保持为说明性文字；资金
  路径值得拥有子句。覆盖率随风险增长，而不是随完整性增长。
- **每个里程碑都能独立成立。** 注册表(registry)、链接器(linker)、DWARF、裁决门(gate)和不安全
  通道(unsafe lane)各自交付独立价值——你可以停在任一层，而你已有的东西
  仍然可用。

Urtext 也与 agent 无关。它不编排任何东西；它定义的是
“运行 oracle(判据) 并交回证据(evidence)”这个协议，任何 AI coding agent
（Claude Code、Codex 等）都可以满足它。它消费 agent；它不
替代 agent。

## 可证伪条件

Urtext 自带它自己的反证，而你应该在投入之前先知道它。
整个系统都押在一个赌注上：

> **为一个子句编写 oracle 的成本，低于逐行评审
> 生成代码的成本。**

如果这个不等式反转，Urtext 就会退化成官僚化的测试编写
仪式——而项目声明的政策是**停止扩张，而不是添加功能
来挽救一个失败的假设。** 内置信号是：每次 `verify`
都会报告该次运行中 **`manual` oracle 的占比**，并在它
超过 50% 时打印警告。注意，该工具只计算*当前运行*的占比——它不会
跟踪历史或“持续”趋势，也不会强制停止；观察趋势
并遵守停止政策，是你的责任。

| 信号 | 它意味着什么 | 诚实的回应 |
|---|---|---|
| 一次运行中的 `manual share` > 50%（观察趋势） | 大多数子句不可判定；赌注可能正在失败 | 停止扩张；不要添加功能来补偿 |
| 绿色 `verify`，但你仍然不信任输出 | oracle 测试错了东西 | 审计 oracle 覆盖；[元审计(meta-audit)](../mechanisms/06-meta-audit-gate.md)就是为此存在的 |
| 子句与代码比例异常上升 | 规范官僚化的早期信号 | 评审粒度；让更多东西保持为说明性文字 |

## 何时*不*该使用 Urtext

Urtext 有意保持狭窄。在以下情况下，请选择别的东西：

- **你需要多 agent 编排、merge queue 或 fleet management。** 那
  不是 Urtext 的工作——像 Gastown 这样的工具覆盖它。Urtext 消费 agent，它
  不协调它们。
- **你想要形式化证明。** oracle 是一种*工程*判断（一个测试、一条
  命令、一个阈值），不是定理。如果你需要 TLA+ 或 Dafny 级保证，
  就使用 TLA+ 或 Dafny。
- **你想要 git 或 CI 的替代品。** Urtext 通过你
  现有的测试和命令运行 oracle；它提供绑定和归因，而不是新的
  构建系统。
- **你的规范稳定、小，而且你的团队纪律良好。** 如果 Spec Kit 的
  约定对你有效，漂移也不是现实问题，那么 enforcement
  layer 可能是你不需要的开销。Urtext 在规范腐烂真的
  咬人之处才物有所值。
- **你需要云协作或多租户工作流。** v0 有意设计为
  单机、单仓库闭环。团队场景（评论、
  并发规范编辑）会被有意推迟，直到出现一位种子用户来
  回答这些问题。

## v0 边界，直说如下

有些能力已经被命名，但**尚未实现**——不要把它们当作
已经存在来规划：

- **`metric` oracle** 在 v0 中会显式失败，而不是评估阈值。
- **视觉和交互 oracle**（针对设计的截图 diff、demo
  replay）是 `oracle` kind 和 `refs` target type 的 v1 扩展。它们
  是原则 P7，不是已发布代码。
- **DWARF range re-anchoring**——当后续编辑移动已映射行时跟随它们——
  是一个标记过的 v0 简化，尚不存在。

Urtext 的纪律是命名这些边界，而不是掩盖它们。一个
假装未实现行为已经存在的工具，正是整个系统要防止的那种
沉默谎言。
