# 从汇编到 C

每个人都会拿出同一个类比：AI 提升抽象层级，就像 C
把抽象层级提升到汇编之上一样。这个类比是对的，但通常的讲法是错的。
C 的胜利并不是因为它“更高层”。它赢是因为**六件事
同时发生了。** 漏掉任何一件，抽象就会泄漏，直到坍缩回
下面那一层。

Urtext 的设计回应了 AI 时代的每一个条件。权威的
[`VISION`](../../../../../zh-CN/VISION.md) 形式化了**四个**承重条件；本页
再加入两个条件（[`DECISIONS D5`](../../../../../zh-CN/DECISIONS.md)），它们在创立时的
讨论中被记录为必要但次要——总计六个。

## 四个承重条件（VISION §2）

| # | 汇编 → C | AI 时代的断裂 | Urtext 的回答 |
|---|---|---|---|
| 1 | 目标层（ISA）先有了形式化语义 | 源语言——自然语言规范——是不可判定的 | 一条规范性陈述必须绑定一个 oracle(判据)；不可验证就意味着错误 |
| 2 | 编译器是确定性的；信任只需建立一次 | LLM 是随机的：同一份规范每次都会产生不同代码 | **确定性从翻译器转移到验证器(verifier)中**——一个生成 → 验证 → 修复循环 |
| 3 | 真相源翻转了：手改编译器输出成了禁忌 | 代码仍被手工编辑，并且永远不会回流到规范 → 规范腐化 | 未映射改动强制约束：一次手工编辑必须回流，或被明确确认 |
| 4 | DWARF 将两层双向缝合起来 | 失败报告一个栈帧，迫使人类回落到代码中 | 设计目标：认知 ↔ 代码 ↔ 证据(evidence) 双向映射，让一次失败归因到一个子句(clause)。v0 交付了子句↔代码存储和手动 `blame`；自动失败到子句归因尚未接线（[DWARF](../mechanisms/05-dwarf-mapping.md)） |

关键的是 **#2。** LLM 承担了过去编译器
承担的“翻译器”角色——但不同于编译器，它是随机的。因此，那种
曾让整个抽象可信的确定性不能再存在于翻译器中。它
必须在某处重建，而唯一剩下的位置就是**验证器。**
所有现有规范工具缺失的正是这一半。

## 两个支撑条件（DECISIONS D5）

另外两个条件没有那么出名，但同样必要——一种抽象如果
跳过它们，仍然会泄漏。

**5. 抽象机器保留成本模型。** C 隐藏了寄存器，但
保留了内存与指针成本的可见性；你仍然可以推理性能。
Urtext 的对应物是：**risk(风险级别) 是子句的一等属性。** 其设计
原则是让延迟、爆炸半径和可逆性保持可见，而不是
被埋起来。v0 将其编码为单一二元分级——`risk:low` 或 `risk:high`——而不是
作为分开的标注维度；更细的成本模型是
这一分级背后的原则，而不是今天语法存储的字段。一种隐藏“这个改动
有多危险”的抽象，比无用还糟。

**6. 存在逃生舱。** `inline asm` 和 `volatile` 存在，是因为约 5% 的
场景需要抽象让位。Urtext 的对应物是：**`risk:high` 和
不安全通道(unsafe lane)。** Urtext 并不假装每一种行为都能被规范化——正
如 C 从未假装内联汇编不存在。在危险路径上，代码
仍是唯一可评审的事实，并退回到人类代码级评审。

## 负面结论

产生这六个条件的同一场讨论，也产生了一个坚定的
否定结论：**伪代码永远不是答案。** 它属于与代码相同的“如何”
类别，它只会增加歧义，并丢弃代码唯一的优点——
可执行性。改变范式意味着改变描述的*对象*
（“必须满足什么”），而不是改变描述的*精度*。

在六个条件都摆上台面之后，最锋利的那一个——规范句子
必须是*可判定*的——值得单独处理。那就是[下一个
概念](03-why-decidable.md)。
