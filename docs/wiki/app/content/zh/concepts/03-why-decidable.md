# 为什么规范必须可判定

这是 Urtext 中其他一切赖以成立的原则，而且它刻意保持
毫不妥协：

> **没有 oracle(判据) 的规范性子句(clause)就是编写错误。**

不是较弱的一类需求。不是“我们稍后再验证”。而是一个错误——
与语法错误同属一类——会让该子句完全无法进入可执行
系统。

## 说明性文字是自由的；子句是受约束的

Urtext 会在你的规范中划出一条硬边界，这条边界就是
**语言与文档**之间的差别。

- **描述性文字是自由的。** 你可以解释上下文、动机、背景、
  权衡——想写多少都可以。它们都不受约束，都不会被
  检查，也都不会进入裁决系统。
- **规范性子句是受约束的。** 一旦某个句子声明系统
  *必须*满足某事，它就必须绑定一个 oracle——一个带有证据、能判定
  通过或失败的检查。没有 oracle 的子句会被记录为 `building` 修订版，
  并且永远无法到达可执行的（`ready`）状态，因此它永远不会运行。（在 v0 中，
  `test`、`cmd` 和 `diff-scope` oracle 会执行；`manual` 是一种产生
  `pending` 的人工检查；`metric` 已声明但尚不可运行——见 [Clauses and
  Oracles](../mechanisms/01-clause-and-oracle.md)。）

从语法上说，这一区分是显式的：子句是一个带有 `C<n>` id 的 Markdown 标题。
没有该 id 的标题就是普通说明文字，
不受任何东西约束。只有你有意提升为子句的陈述才会进入
判定系统。你永远不会被迫把一切都规范化——你只是
被迫让你*确实*规范化的内容可检查。

## “可判定”带来了什么

一旦规范性陈述必须携带 oracle，传统规范驱动开发中的四类失败模式
就会从源头被关闭：

- **没有不可检查的裁决门(gate)。** 语法不会评判文字——你*可以*写
  `## C001 It should be fast <!-- oracle:cmd:true -->`，它也会被索引。语法
  拒绝的是*缺失或无效的* oracle：没有 oracle 的子句永远
  到不了 `ready`。把“fast”变成真正的阈值（一个像 `metric` 表达式
  `p99 < 200ms`，或测量它的 `cmd`/`test`）是一种编写纪律，语法会推动它
  但不会强制；作弊的 oracle 则交给
  [元审计(meta-audit)](../mechanisms/06-meta-audit-gate.md) 处理。
- **没有自我打分。** 完成度是客观证据的聚合——
  通过率(pass rate)是绿色可运行子句占*已判定*可运行子句的比例——而不是
  AI 给自己工作分配的数字。
- **没有沉默的空缺。** 被声明为子句却没有 oracle 的需求不会
  被悄悄推迟；它会在编写时以 `missing_oracle` 的形式被响亮拒绝。
- **手动归因到子句。** 当检查失败时，`urtext blame` 会把
  失败行映射回约束它的子句——v0 中这是人工查找，
  还不是自动报告（见 [DWARF](../mechanisms/05-dwarf-mapping.md)）。

## oracle 与测试并不相同

一个常见误读是：“所以这只是多了几个步骤的 TDD。”不是。由
写实现的同一个 agent 写出的测试是**同源验证**——红灯或
绿灯只证明 AI 对规范的理解是*自洽的*，并不证明
标准已经*被满足*。两个模型可能对同一个误读达成一致。

oracle 比单元测试更宽泛，而且其*意图*是一个**独立的
第三方事实标准**——一种同时独立于规范措辞和实现选择的检查。
Urtext 不强制这种独立性：它记录 oracle 的 kind 和 ref，并重新运行它，
但它无法知道测试是谁编写的，也无法知道它是否真正独立。工具保证的是
*客观且可重跑*的证据；让 oracle 保持真正独立是一种
编写纪律，由元审计支撑（而非取代）。Urtext
定义了五种 oracle kind（`test`、`cmd`、`metric`、`diff-scope`、`manual`），
而对抗性评审被保留给一个*元*层，用来审计
证据是否真正覆盖子句——绝不作为证据
本身的替代品。这些机制详见
[Clauses and
Oracles](../mechanisms/01-clause-and-oracle.md) 和 [Meta-Audit and the
Gate](../mechanisms/06-meta-audit-gate.md)。

## 承重赌注

可判定性不是免费的。为一个子句编写 oracle 是有成本的，而
Urtext 把自身存在押在一个单一赌注上：

> **为一个子句编写 oracle 的成本，低于逐行评审
> 由此生成的代码的成本。**

如果这个不等式反转，Urtext 就会退化为官僚式的
测试编写仪式——而它会把这一点明说出来。系统自带自己的
证伪条件：`manual` oracle 的占比(manual share)是一个健康指标，而
持续占多数意味着这个赌注已经失败。一个携带自身
反证的系统是工程；一个无法被证伪的系统是信仰。参见 [Adoption and
Limits](../guides/05-adoption-and-limits.md)，了解这个绊线如何工作。
